Kubernetes em Produção

Este guia documenta o fluxo completo de implementação de aplicações em Kubernetes para ambiente de produção, cobrindo desde o build da imagem Docker até a configuração de storage persistente.

Fluxo de Implementação​

1

Build da Imagem Docker

Construção da imagem da aplicação com as boas práticas de otimização para produção, incluindo multi-stage builds e uso correto de .dockerignore.

2

Push da Imagem

Envio da imagem para um registry centralizado. São abordadas duas opções: GitLab Container Registry e Registry Local (self-hosted).

3

Configuração de Storage no Kubernetes

Provisionamento de volumes persistentes com PersistentVolume, PersistentVolumeClaim e StorageClass para garantir durabilidade dos dados.

Pré-requisitos​

Antes de iniciar, certifique-se que os seguintes recursos estão disponíveis:

docker
Docker

Docker Engine instalado e funcionando na máquina de build. Versão recomendada: 24.x ou superior.

terminal
kubectl

CLI do Kubernetes configurado com acesso ao cluster de produção via kubeconfig.

lock
Acesso ao Registry

Credenciais configuradas para o GitLab Container Registry ou para o registry local self-hosted.

server
Cluster Kubernetes

Cluster Kubernetes funcional com permissões para criar recursos como PV, PVC e StorageClass.
