CyberArk Conjur no Kubernetes

O CyberArk Conjur é uma plataforma enterprise de gerenciamento de segredos com foco em controle de acesso granular baseado em políticas (RBAC declarativo), auditoria completa e integração nativa com ambientes Kubernetes. É amplamente adotado em ambientes corporativos que já utilizam o ecossistema CyberArk (PAM).

Arquitetura de Integração​

GitLab CI/CD ──► JWT/API Key Auth ──► Conjur ──► segredos na pipeline
                                         │
Kubernetes Pod ──► Conjur Authenticator ──► Conjur ──► segredos via:
                   (Kubernetes Auth)           ├── Secrets Provider (init container)
                                               ├── Secretless Broker (sidecar)
                                               └── External Secrets Operator

Pré-requisitos​

  • Conjur OSS ou Conjur Enterprise em execução (on-premise ou cloud)
  • conjur-cli instalado localmente
  • Acesso de admin ao Conjur para criar políticas

Instalação do Conjur OSS no Kubernetes​

helm repo add cyberark https://cyberark.github.io/helm-charts
helm repo update

helm install conjur cyberark/conjur-oss \
  --namespace conjur \
  --create-namespace \
  --set dataKey="$(openssl rand -base64 32)" \
  --set authenticators="authn\,authn-k8s/producao-cluster" \
  --set service.external.enabled=true

Inicializar o Conjur​

# Obter o pod do Conjur
CONJUR_POD=$(kubectl get pod -n conjur -l app=conjur-oss -o jsonpath='{.items[0].metadata.name}')

# Criar conta e admin
kubectl exec -n conjur $CONJUR_POD -- conjurctl account create mycompany
# Saída: API key for admin: <GUARDAR-ESTE-TOKEN>

# Configurar CLI
conjur init -u https://conjur.empresa.com.br -a mycompany
conjur login -i admin -p <API-KEY-DO-ADMIN>

Modelo de Políticas do Conjur​

O Conjur usa YAML declarativo para definir o modelo de acesso. Tudo é um recurso com políticas.

Estrutura de políticas para Kubernetes​

# policy-root.yml — política raiz
- !policy
  id: producao
  body:
    # ── Segredos da aplicação ────────────────────────────────────
    - !policy
      id: minha-app
      body:
        - !variable db/password
        - !variable db/username
        - !variable api/key
        - !variable jwt/secret

    # ── Hosts (identidades dos workloads) ────────────────────────
    - !policy
      id: kubernetes
      body:
        - !policy
          id: apps
          body:
            # Identidade do Pod no Kubernetes
            - !layer

            - !host
              id: minha-app/@@/producao
              annotations:
                authn-k8s/namespace: producao
                authn-k8s/service-account: minha-app-sa
                authn-k8s/deployment: minha-app
                authn-k8s/authentication-container-name: conjur-authenticator

    # ── Permissões: liga hosts aos segredos ──────────────────────
    - !grant
      role: !layer producao/kubernetes/apps
      member: !host producao/kubernetes/apps/minha-app/@@/producao

    - !permit
      role: !layer producao/kubernetes/apps
      privileges: [read, execute]
      resources:
        - !variable producao/minha-app/db/password
        - !variable producao/minha-app/db/username
        - !variable producao/minha-app/api/key
        - !variable producao/minha-app/jwt/secret
# Carregar a política no Conjur
conjur policy load -b root -f policy-root.yml

# Definir os valores dos segredos
conjur variable set -i producao/minha-app/db/password -v "senha-super-secreta"
conjur variable set -i producao/minha-app/db/username -v "app_user"
conjur variable set -i producao/minha-app/api/key     -v "chave-api-externa-123"
conjur variable set -i producao/minha-app/jwt/secret  -v "meu-jwt-secret-producao"

Configurar o Autenticador Kubernetes​

O Conjur precisa de permissão para verificar tokens de ServiceAccount do Kubernetes:

# Habilitar o autenticador Kubernetes no Conjur
kubectl exec -n conjur $CONJUR_POD -- \
  conjurctl authenticator enable --authenticator-id authn-k8s/producao-cluster

# Criar os recursos necessários no cluster
kubectl create namespace conjur
kubectl create serviceaccount conjur-authn-sa --namespace=conjur

# ClusterRole para o Conjur revisar tokens
kubectl apply -f - <<EOF
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: conjur-authenticator
rules:
  - apiGroups: [""]
    resources: ["pods", "serviceaccounts"]
    verbs: ["get", "list"]
  - apiGroups: ["extensions", "apps"]
    resources: ["deployments", "replicasets"]
    verbs: ["get", "list"]
  - apiGroups: [""]
    resources: ["pods/exec"]
    verbs: ["create", "get"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: conjur-authenticator
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: conjur-authenticator
subjects:
  - kind: ServiceAccount
    name: conjur-authn-sa
    namespace: conjur
EOF

# Pegar certificado do cluster
CLUSTER_CA=$(kubectl config view --raw --minify --flatten \
  -o jsonpath='{.clusters[].cluster.certificate-authority-data}' | base64 -d)

# Pegar URL do API Server
API_URL=$(kubectl config view --minify -o jsonpath='{.clusters[].cluster.server}')

# Criar ConfigMap para os apps saberem como autenticar
kubectl apply -f - <<EOF
apiVersion: v1
kind: ConfigMap
metadata:
  name: conjur-connect
  namespace: producao
data:
  CONJUR_APPLIANCE_URL: "https://conjur.empresa.com.br"
  CONJUR_ACCOUNT: "mycompany"
  CONJUR_AUTHN_URL: "https://conjur.empresa.com.br/authn-k8s/producao-cluster"
  CONJUR_SSL_CERTIFICATE: |
$(echo "$CLUSTER_CA" | sed 's/^/    /')
EOF

Método 1: Secrets Provider (Init Container)​

O Conjur Secrets Provider roda como init container, busca os segredos antes da aplicação iniciar e os disponibiliza como arquivos ou variáveis de ambiente:

# Criar ServiceAccount da aplicação
kubectl create serviceaccount minha-app-sa --namespace=producao
# deployment-conjur-init.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: minha-app
  namespace: producao
spec:
  replicas: 2
  selector:
    matchLabels:
      app: minha-app
  template:
    metadata:
      labels:
        app: minha-app
    spec:
      serviceAccountName: minha-app-sa

      # ── Init Container: Secrets Provider ──────────────────────
      initContainers:
        - name: conjur-secrets-provider
          image: cyberark/secrets-provider-for-k8s:latest
          imagePullPolicy: Always
          env:
            - name: MY_POD_NAME
              valueFrom:
                fieldRef:
                  fieldPath: metadata.name
            - name: MY_POD_NAMESPACE
              valueFrom:
                fieldRef:
                  fieldPath: metadata.namespace
          envFrom:
            - configMapRef:
                name: conjur-connect             # ConfigMap criado acima
          env:
            - name: CONJUR_AUTHN_LOGIN
              value: "host/producao/kubernetes/apps/minha-app/@@/producao"
            - name: SECRETS_DESTINATION
              value: "k8s_secrets"               # Grava em Kubernetes Secrets
            # Alternativa: "file" para gravar em /conjur/secrets/
          volumeMounts:
            - name: podinfo
              mountPath: /conjur/podinfo
          resources:
            requests:
              cpu: "50m"
              memory: "64Mi"

      # ── Container principal ─────────────────────────────────────
      containers:
        - name: minha-app
          image: registry.gitlab.com/empresa/minha-app:1.0.0
          envFrom:
            - secretRef:
                name: minha-app-conjur-secrets   # Secret preenchido pelo init container
          ports:
            - containerPort: 3000
          resources:
            requests:
              cpu: "250m"
              memory: "256Mi"
            limits:
              cpu: "500m"
              memory: "512Mi"

      volumes:
        - name: podinfo
          downwardAPI:
            items:
              - path: "annotations"
                fieldRef:
                  fieldPath: metadata.annotations

SecretTemplate — mapear variáveis Conjur para Kubernetes Secret​

# conjur-secrettemplate.yaml
apiVersion: v1
kind: Secret
metadata:
  name: minha-app-conjur-secrets
  namespace: producao
  annotations:
    # Mapeamento: chave-no-secret = caminho-no-conjur
    conjur.org/conjur-secrets: |
      DB_PASSWORD: producao/minha-app/db/password
      DB_USERNAME: producao/minha-app/db/username
      API_KEY: producao/minha-app/api/key
      JWT_SECRET: producao/minha-app/jwt/secret
    conjur.org/secret-file-format: "dotenv"
type: Opaque

Método 2: Secretless Broker (Sidecar Proxy)​

O Secretless Broker intercepta conexões da aplicação e injeta as credenciais transparentemente. A aplicação não precisa saber das credenciais:

# deployment-secretless.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: minha-app-secretless
  namespace: producao
spec:
  replicas: 2
  selector:
    matchLabels:
      app: minha-app
  template:
    metadata:
      labels:
        app: minha-app
    spec:
      serviceAccountName: minha-app-sa
      containers:
        - name: minha-app
          image: registry.gitlab.com/empresa/minha-app:1.0.0
          env:
            # A aplicação conecta no broker local, sem precisar de senha
            - name: DATABASE_URL
              value: "postgres://localhost:5432/producao"
          ports:
            - containerPort: 3000

        # ── Sidecar: Secretless Broker ──────────────────────────
        - name: secretless-broker
          image: cyberark/secretless-broker:latest
          args: ["-f", "/etc/secretless/config.yml"]
          envFrom:
            - configMapRef:
                name: conjur-connect
          env:
            - name: CONJUR_AUTHN_LOGIN
              value: "host/producao/kubernetes/apps/minha-app/@@/producao"
            - name: MY_POD_NAME
              valueFrom:
                fieldRef:
                  fieldPath: metadata.name
            - name: MY_POD_NAMESPACE
              valueFrom:
                fieldRef:
                  fieldPath: metadata.namespace
          volumeMounts:
            - name: secretless-config
              mountPath: /etc/secretless
              readOnly: true
            - name: podinfo
              mountPath: /conjur/podinfo

      volumes:
        - name: secretless-config
          configMap:
            name: secretless-broker-config
        - name: podinfo
          downwardAPI:
            items:
              - path: "annotations"
                fieldRef:
                  fieldPath: metadata.annotations
# secretless-config.yaml — configuração do proxy
apiVersion: v1
kind: ConfigMap
metadata:
  name: secretless-broker-config
  namespace: producao
data:
  config.yml: |
    listeners:
      - name: pg-listener
        protocol: pg
        address: "localhost:5432"

    handlers:
      - name: pg-handler
        listener: pg-listener
        credentials:
          - name: address
            provider: literal
            id: "postgres-service.producao.svc.cluster.local:5432"
          - name: username
            provider: conjur
            id: producao/minha-app/db/username
          - name: password
            provider: conjur
            id: producao/minha-app/db/password
          - name: dbname
            provider: literal
            id: producao

Método 3: External Secrets Operator com Conjur​

# conjur-secretstore.yaml
apiVersion: external-secrets.io/v1beta1
kind: SecretStore
metadata:
  name: conjur-backend
  namespace: producao
spec:
  provider:
    conjur:
      url: "https://conjur.empresa.com.br"
      caBundle: "<BASE64-DO-CERTIFICADO-CA>"
      auth:
        kubernetes:
          account: "mycompany"
          serviceAccountRef:
            name: minha-app-sa
          authenticatorID: "producao-cluster"
          hostID: "host/producao/kubernetes/apps/minha-app/@@/producao"
# externalsecret-conjur.yaml
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: minha-app-secrets
  namespace: producao
spec:
  refreshInterval: 5m
  secretStoreRef:
    name: conjur-backend
    kind: SecretStore
  target:
    name: minha-app-conjur-sync
    creationPolicy: Owner
  data:
    - secretKey: DB_PASSWORD
      remoteRef:
        key: producao/minha-app/db/password
    - secretKey: DB_USERNAME
      remoteRef:
        key: producao/minha-app/db/username
    - secretKey: API_KEY
      remoteRef:
        key: producao/minha-app/api/key
    - secretKey: JWT_SECRET
      remoteRef:
        key: producao/minha-app/jwt/secret

Integração com GitLab CI/CD​

Configurar política para pipelines GitLab​

# policy-gitlab-ci.yml
- !policy
  id: gitlab-ci
  body:
    # Segredos acessíveis pela pipeline
    - !variable ci/registry-password
    - !variable ci/deploy-token
    - !variable ci/sonar-token

    # Identidade da pipeline GitLab
    - !host
      id: gitlab/minha-app
      annotations:
        description: "Pipeline GitLab do projeto minha-app"

    - !layer

    - !grant
      role: !layer
      member: !host gitlab/minha-app

    - !permit
      role: !layer
      privileges: [read, execute]
      resources:
        - !variable gitlab-ci/ci/registry-password
        - !variable gitlab-ci/ci/deploy-token
        - !variable gitlab-ci/ci/sonar-token
        # Acesso somente leitura aos segredos de produção (para deploy)
        - !variable producao/minha-app/db/password
        - !variable producao/minha-app/api/key
# Carregar política e definir valor inicial da API key da pipeline
conjur policy load -b root -f policy-gitlab-ci.yml

# Gerar API key para o host da pipeline
API_KEY=$(conjur host rotate-api-key --id gitlab-ci/gitlab/minha-app)
echo "API key gerada: $API_KEY"
# Armazene esse valor como CI/CD variable no GitLab: CONJUR_AUTHN_API_KEY

No projeto GitLab, em Settings → CI/CD → Variables, configure:

.gitlab-ci.yml — Buscar segredos do Conjur via API key​

# .gitlab-ci.yml
stages:
  - test
  - build
  - deploy

variables:
  CONJUR_APPLIANCE_URL: "https://conjur.empresa.com.br"
  CONJUR_ACCOUNT: "mycompany"

# ── Template reutilizável de autenticação no Conjur ───────────────
.conjur-auth: &conjur-auth
  before_script:
    - apk add --no-cache curl jq
    - |
      # Autenticar com API key e obter session token
      CONJUR_SESSION_TOKEN=$(curl -sf \
        --request POST \
        --data "${CONJUR_AUTHN_API_KEY}" \
        "${CONJUR_APPLIANCE_URL}/authn/${CONJUR_ACCOUNT}/${CONJUR_AUTHN_LOGIN}/authenticate" \
        | base64)

      if [ -z "$CONJUR_SESSION_TOKEN" ]; then
        echo "Erro: falha ao autenticar no Conjur"
        exit 1
      fi

      # Função auxiliar para buscar segredos
      get_secret() {
        local SECRET_ID="$1"
        curl -sf \
          -H "Authorization: Token token=\"${CONJUR_SESSION_TOKEN}\"" \
          "${CONJUR_APPLIANCE_URL}/secrets/${CONJUR_ACCOUNT}/variable/$(python3 -c "import urllib.parse; print(urllib.parse.quote('${SECRET_ID}', safe=''))")"
      }

      # Buscar segredos necessários
      export DB_PASSWORD=$(get_secret "producao/minha-app/db/password")
      export API_KEY=$(get_secret "producao/minha-app/api/key")
      export REGISTRY_PASSWORD=$(get_secret "gitlab-ci/ci/registry-password")

      echo "Segredos carregados do Conjur com sucesso"

# ── Usando conjur-cli (alternativa mais robusta) ──────────────────
.conjur-auth-cli: &conjur-auth-cli
  image: cyberark/conjur-cli:8
  before_script:
    - |
      conjur init \
        --url "${CONJUR_APPLIANCE_URL}" \
        --account "${CONJUR_ACCOUNT}" \
        --self-signed   # Remover em produção com CA válida

      conjur login \
        --id "${CONJUR_AUTHN_LOGIN}" \
        --password "${CONJUR_AUTHN_API_KEY}"

      # Exportar segredos como variáveis de ambiente
      export DB_PASSWORD=$(conjur variable get -i "producao/minha-app/db/password")
      export API_KEY=$(conjur variable get -i "producao/minha-app/api/key")

# ── Jobs da pipeline ──────────────────────────────────────────────
unit-tests:
  stage: test
  image: node:20-alpine
  <<: *conjur-auth
  script:
    - npm ci
    - npm test
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
    - if: $CI_COMMIT_BRANCH == "main"

build-image:
  stage: build
  image: docker:24
  services:
    - docker:24-dind
  variables:
    DOCKER_TLS_CERTDIR: "/certs"
  before_script:
    - apk add --no-cache curl jq
    - |
      # Autenticar no Conjur
      CONJUR_SESSION_TOKEN=$(curl -sf \
        --request POST \
        --data "${CONJUR_AUTHN_API_KEY}" \
        "${CONJUR_APPLIANCE_URL}/authn/${CONJUR_ACCOUNT}/${CONJUR_AUTHN_LOGIN}/authenticate" \
        | base64)

      # Buscar senha do registry privado do Conjur
      REGISTRY_PASS=$(curl -sf \
        -H "Authorization: Token token=\"${CONJUR_SESSION_TOKEN}\"" \
        "${CONJUR_APPLIANCE_URL}/secrets/${CONJUR_ACCOUNT}/variable/gitlab-ci%2Fci%2Fregistry-password")

      docker login registry.empresa.local:5000 \
        --username deploy \
        --password "${REGISTRY_PASS}"
  script:
    - docker build -t $CI_REGISTRY_IMAGE:$CI_COMMIT_SHORT_SHA .
    - docker push $CI_REGISTRY_IMAGE:$CI_COMMIT_SHORT_SHA
  rules:
    - if: $CI_COMMIT_BRANCH == "main"

deploy-producao:
  stage: deploy
  image: bitnami/kubectl:latest
  <<: *conjur-auth
  script:
    - |
      # Atualizar Kubernetes Secret com valores frescos do Conjur
      kubectl create secret generic minha-app-secrets \
        --from-literal=DB_PASSWORD="${DB_PASSWORD}" \
        --from-literal=API_KEY="${API_KEY}" \
        --namespace=producao \
        --dry-run=client -o yaml | kubectl apply -f -

      kubectl apply -f k8s/ --namespace=producao
      kubectl rollout status deployment/minha-app --namespace=producao --timeout=5m
  environment:
    name: producao
    url: https://minha-app.empresa.com.br
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
      when: manual

.gitlab-ci.yml — Integração com Conjur JWT Auth (sem API key estática)​

Para maior segurança, use JWT para evitar API keys estáticas na pipeline:

# Configurar JWT Auth no Conjur
conjur policy load -b root -f - <<EOF
- !policy
  id: conjur/authn-jwt/gitlab
  body:
    - !webservice

    - !variable jwks-uri
    - !variable token-app-property
    - !variable issuer

    - !group consumers

    - !permit
      role: !group consumers
      privilege: [read, authenticate]
      resource: !webservice
EOF

# Configurar valores
conjur variable set -i conjur/authn-jwt/gitlab/jwks-uri \
  -v "https://gitlab.empresa.com.br/-/jwks"
conjur variable set -i conjur/authn-jwt/gitlab/token-app-property \
  -v "project_path"
conjur variable set -i conjur/authn-jwt/gitlab/issuer \
  -v "https://gitlab.empresa.com.br"
# .gitlab-ci.yml — com JWT (sem API key!)
deploy-producao-jwt:
  stage: deploy
  image: bitnami/kubectl:latest
  id_tokens:
    CONJUR_JWT_TOKEN:
      aud: "conjur"              # audience deve bater com a configuração do Conjur
  before_script:
    - |
      # Trocar o JWT por um session token do Conjur
      CONJUR_SESSION_TOKEN=$(curl -sf \
        --request POST \
        --header "Content-Type: application/x-www-form-urlencoded" \
        --data "jwt=${CONJUR_JWT_TOKEN}" \
        "${CONJUR_APPLIANCE_URL}/authn-jwt/gitlab/${CONJUR_ACCOUNT}/authenticate" \
        | base64)

      export DB_PASSWORD=$(curl -sf \
        -H "Authorization: Token token=\"${CONJUR_SESSION_TOKEN}\"" \
        "${CONJUR_APPLIANCE_URL}/secrets/${CONJUR_ACCOUNT}/variable/producao%2Fminha-app%2Fdb%2Fpassword")
  script:
    - kubectl apply -f k8s/ --namespace=producao
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
      when: manual

Auditoria e Controle​

O Conjur registra todas as operações. Consulte o audit log:

# Ver audit log via API
curl -sf \
  -H "Authorization: Token token=\"${CONJUR_SESSION_TOKEN}\"" \
  "${CONJUR_APPLIANCE_URL}/audit/events?limit=50" | jq '.'

# Filtrar por recurso específico
curl -sf \
  -H "Authorization: Token token=\"${CONJUR_SESSION_TOKEN}\"" \
  "${CONJUR_APPLIANCE_URL}/audit/events?resource_id=mycompany:variable:producao/minha-app/db/password"
